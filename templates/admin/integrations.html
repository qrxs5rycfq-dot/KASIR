Panduan Integrasi

Penting!

GrabFood, GoFood, dan ShopeeFood menggunakan sistem Merchant Integration yang memerlukan pendaftaran sebagai mitra resmi. Setelah terdaftar, Anda akan mendapatkan akses untuk mengkonfigurasi webhook yang akan mengirim pesanan otomatis ke sistem kasir ini.

GrabFood

  1. 1. Daftar sebagai merchant di GrabMerchant Portal
  2. 2. Akses menu Developer Settings
  3. 3. Masukkan Webhook URL di atas ke kolom Order Notification URL
  4. 4. Salin Webhook Secret dari portal
  5. 5. Signature dikirim via header X-Grab-Signature (HMAC-SHA256)
  6. 6. Set environment variable:
    GRABFOOD_WEBHOOK_SECRET=secret_anda

GoFood

  1. 1. Daftar sebagai mitra di GoBiz Dashboard
  2. 2. Akses API Integration settings
  3. 3. Masukkan Webhook URL di atas ke kolom notification URL
  4. 4. Salin API Secret dari dashboard
  5. 5. Signature dikirim via header X-Callback-Token (HMAC-SHA512)
  6. 6. Set environment variable:
    GOFOOD_WEBHOOK_SECRET=secret_anda

ShopeeFood

  1. 1. Daftar sebagai merchant di Shopee Partner Portal
  2. 2. Akses Webhook Configuration
  3. 3. Masukkan Webhook URL di atas ke kolom Order Webhook
  4. 4. Salin Webhook Secret dari portal
  5. 5. Signature dikirim via header X-Shopee-Hmac-SHA256 (HMAC-SHA256)
  6. 6. Set environment variable:
    SHOPEEFOOD_WEBHOOK_SECRET=secret_anda

Environment Variables

Tambahkan variable berikut di file .env atau environment server Anda:

# GrabFood Integration (HMAC-SHA256 via X-Grab-Signature)
GRABFOOD_WEBHOOK_SECRET=your_grabfood_secret_here
GRABFOOD_STORE_ID=your_store_id
GRABFOOD_STORE_BRANCH_MAP=store123:PUSAT,store456:CBG-S

# GoFood Integration (HMAC-SHA512 via X-Callback-Token)
GOFOOD_WEBHOOK_SECRET=your_gofood_secret_here
GOFOOD_STORE_ID=your_store_id
GOFOOD_STORE_BRANCH_MAP=outlet789:PUSAT

# ShopeeFood Integration (HMAC-SHA256 via X-Shopee-Hmac-SHA256)
SHOPEEFOOD_WEBHOOK_SECRET=your_shopeefood_secret_here
SHOPEEFOOD_STORE_ID=your_store_id
SHOPEEFOOD_STORE_BRANCH_MAP=shop111:PUSAT

Deferred Printing

Pesanan dari webhook tidak langsung dicetak saat diterima. Pesanan masuk ke antrian print (PendingPrint) dan akan diambil oleh Printer Station secara periodik. Ini mencegah timeout dan duplikasi cetak jika platform melakukan retry.

Status alur: Received → Accepted → Printed

Multi-Cabang (Store ID Mapping)

Untuk multi-cabang, konfigurasi *_STORE_BRANCH_MAP environment variable dengan format store_id:BRANCH_CODE. Platform mengirim store/outlet ID dalam payload, dan sistem akan otomatis merutekan ke cabang yang benar. Jika tidak ada mapping yang cocok, pesanan masuk ke cabang Pusat.
